# 麻酔科研修医ドリル MVP

教育病院での研修開始前学習を想定した、ブラウザ実行型の計算ゲームMVPです。

## できること

- トップページで受講者 **A〜E** から選択して入室
- コンテンツ一覧から「γ計算ゲーム」を選択
- セッション開始時に **5問モード** / **10問モード** を選択（毎回変えられる）
- 基本編（レミフェンタニル 5mg/50ml）
- 応用編（レミフェンタニル + ノルアドレナリン 3mg/50ml, 5mg/50ml）
- 「コンテンツは拡張中」を表示
- ゲーム開始後は **全画面**（一覧とは別レイヤ）
- **イントロ** で **体重** と **薬剤（組成）** を大きく表示 → **GO** で **指導医の指示（γ）** → **ml/h** で速答
- **レベル1は1問30秒**。ブロックが進むほどレベルが上がり、制限時間が段階的に短くなる（最短10秒）
- **5問単位で体重が固定**（イントロで更新。出題中は「このブロック共通」と表示）
- 画面上の **電卓** で暗算なし（結果は「答えに反映」）
- 時間切れは不正解扱いで自動で次の問題へ
- **個人別**にログ・ランキングをローカル保存（`A`〜`E` ごとに別ストレージ）
- セッション中 **残り◯問** をヘッダー・イントロに表示
- **正答率**はセッション終了時に大表示（**ログに登録**を押したときだけ個人ログに保存）
- 各問の正誤は **大きなオーバーレイ** で短時間表示（次の解答画面には残さない）
- イントロの **GO** は **Enter** でも開始可能
- 指導医ダッシュボード（選択中の受講者のログのみ）

## Vercel へのデプロイ（本番ホスト想定）

今後コンテンツや API を足していく前提で、**Vercel 上にホスト**できるようにしています。現状は **ビルド不要の静的サイト**（ルートの `index.html` / `styles.css` / `app.js`）です。

### 1. Git に載せる（GitHub 連携のため）

```bash
cd masui.introduction
git init
git add .
git commit -m "Initial commit"
```

GitHub に空リポジトリを作り、`git remote add` → `git push` してください。

### 2. Vercel でプロジェクト作成（推奨: ダッシュボード）

1. [vercel.com](https://vercel.com) にログイン → **Add New…** → **Project** → リポジトリを **Import**
2. **Framework Preset**: `Other`（または自動検出のまま）
3. **Root Directory**: `./`（そのまま）
4. **Build Command**: 空のまま
5. **Output Directory**: 空のまま
6. **Deploy** → 完了後、**Domains** に `https://＜プロジェクト名＞.vercel.app` が表示されます

以降は `main`（または設定したブランチ）へ **push するたびに自動デプロイ**されます。PR には **Preview URL** が付きます。

### 3. CLI で直接デプロイする場合

```bash
cd /path/to/masui.introduction
npx vercel login
npx vercel          # プレビュー
npx vercel --prod   # 本番
```

`.vercel` は `.gitignore` に含めています。

### 拡充時のメモ（Vercel）

| 追加するもの | 目安 |
|-------------|------|
| 画像・データ JSON など | ルートまたは `public/` を切って配置し、パスを HTML/JS から参照 |
| **Serverless API** | リポジトリ直下に `api/*.js`（Node）を置くと Vercel がルート `/api/...` として認識 |
| **フレームワーク**（Vite / Next 等） | そのツールのビルド出力先を Vercel の **Output Directory** に合わせる |
| 環境変数 | Vercel プロジェクトの **Settings → Environment Variables** に登録（クライアントに載せる値は `NEXT_PUBLIC_` 等のルールに従う） |

設定ファイルはルートの `vercel.json` です（ヘッダー等はここに追記可能）。

## 起動方法

1. `index.html` をブラウザで開く
2. 受講者を選び `入室する` を押す
3. `ゲーム開始（基本編 or 応用編）` → **5問** または **10問** を選択
4. イントロで体重・薬剤を確認し `GO` → 指示に答える（Enterでも回答確定）

## スコア仕様（MVP）

- 正答: +100
- 誤答: -30
- ノルアドレナリン問題の誤答: 追加 -30
- 3連続正答ボーナス: +40
- 応用編では時間補正あり
- 制限時間を超えると不正解で次問題へ自動進行

## 計算式

- `ml/h = γ(μg/kg/min) × 体重(kg) × 60 ÷ 濃度(μg/ml)`
- 判定は小数第1位で比較（±0.1 ml/h 以内を正解）

## ログ保存（ローカル）

- 受講者ごとに `localStorage` に保存（例: A なら `masui_attempts_A`）
- **セッション終了後、「ログに登録」を押したセッションだけ**が保存される（押さずに一覧へ戻ると保存されない）
- 保存内容: スコア・正答率・全回答など
- ダッシュボードは入室中の受講者の**保存済み**ログのみ集計
